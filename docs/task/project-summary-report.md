# Primitive-CUA 课题成果汇报

## 一、项目情况总结

### 1.1 项目概述

**Primitive-CUA** 是一个面向 Windows 桌面环境的 AI 操控代理（Computer Use Agent），基于 Moonshot Kimi K3 多模态大模型构建。项目实现了"看到屏幕 → 理解任务 → 操控桌面 → 完成任务"的完整闭环，支持三种执行模式：全 AI Agent、快速回放、脚本引擎。

### 1.2 项目规模

| 维度 | 数据 |
|------|------|
| 开发周期 | 9 天（2026/7/15 — 7/24） |
| 代码提交 | 194 次 |
| Python 源文件 | 40 个 |
| 代码总行数 | ~12,800 行 |
| 工具/能力 | 49 个（截图、键鼠、OCR、浏览器、UIA、Shell、PDF 等） |
| 知识库 | 10 个领域知识文件 + 16 个自动学习技能 |
| 文档 | 6 份技术文档（API 参考、脚本引擎、宏指南、元件引擎等） |

### 1.3 核心能力矩阵

```
                     Agent 模式     快速回放      脚本引擎
                     ─────────     ────────      ────────
  执行速度           ~5s/步        10ms-5s/步    10ms/步
  AI 依赖            全依赖         L0/L1 不依赖   不依赖*
  分支/循环          内置          无             完整 DSL
  跨任务记忆         ✓ (ChromaDB)  无             无
  自我改进           ✓ (AutoSkill)  ✓ (L2自愈)    ✓ (K3脚本生成)
  适用场景           探索/复杂任务  重复性操作     结构化流程
```

\* 除 `kimi`/`draft`/`genimg` 命令外不依赖 LLM

## 二、项目开发过程总结

### 2.1 开发阶段

**第一阶段：原型搭建（7/15-7/16）**
- 搭建项目骨架、配置 Kimi K3 API 对接
- 实现基础工具：截图（mss+overlay）、鼠标操控（归一化坐标）、键盘输入、OCR（RapidOCR）
- 完成 Agent 核心循环：截图→推理→工具调用→验证→再推理

**第二阶段：能力扩展（7/16-7/19）**
- Playwright 浏览器工具（14 个工具）
- Windows UIA 自动化（5 个工具）
- 窗口管理、剪贴板、文件操作
- 子代理：DraftContent（内容写作）、GenerateImage（SVG 生成）
- 学习系统：四层架构（AutoSkill + Reflection + Pending + Knowledge）
- 知识库系统：ChromaDB 向量索引 + 手动知识文件

**第三阶段：录制回放与脚本引擎（7/19-7/22）**
- 元件引擎：截屏→裁剪→OCR→dHash→嵌入→存储
- 宏录制/回放：L0 像素匹配 → L1 嵌入匹配 → L2 K3 自愈
- .cua 脚本 DSL：变量/条件/循环/重试/异常捕获
- 元件管理器：拖拽框选、添加/删除/重命名/测试

**第四阶段：智能化与鲁棒性（7/22-7/24）**
- K3 脚本决策引擎：Top3 候选 → K3 选择/创建/拒绝
- 多语言嵌入模型（中英双语 MiniLM-L12，离线加载）
- 脚本扩展：12 个 Web 命令、Shell、Ask、Draft、Genimg、Kimi 接力
- PDF 报告生成、任务计时、窗口优先模板匹配
- 上下文自愈：孤儿 tool_calls 双向修复
- 统一数据目录 + --repair 向量索引重建

## 四、项目亮点总结

### 4.1 三层执行架构

独创的 **Agent / 快速回放 / 脚本引擎** 三层体系，同一任务可按复杂度选择最优执行模式：

- **探索性任务** → K3 Agent（全 AI，灵活适应）
- **重复性操作** → 快速回放（L0 像素 <10ms → L1 嵌入 ~100ms → L2 K3 自愈 ~5s）
- **结构化流程** → 脚本引擎（文本 DSL，有分支/循环/变量，10ms/步）

### 4.2 K3 自主决策引擎

执行前 K3 审查 Top3 候选脚本的完整内容，检查元件库是否存在对应元素（缺失则警告），自主选择：
- **执行** 现有脚本
- **创建** 新脚本（自动验证 → 自愈修复 → 保存）
- **拒绝** 所有候选项，退回普通 Agent 模式

### 4.3 脚本 DSL 与 AI 接力

自研 `.cua` 领域特定语言，支持 40+ 命令，包括：
- 12 个 Web 自动化命令 + `if web`/`if url` 条件
- `kimi` 中途 AI 接力（截图→K3 操作→结果回传→脚本继续）
- `shell` 命令行、`draft` 内容生成、`name` 智能命名
- try/catch/retry 自愈模式

脚本失败后自动交接：`return 2` → 完整剧本源码 + 进度日志 + 截图 → K3 继续完成任务

### 4.4 多语言离线嵌入 + 自愈

- 多语言模型 MiniLM-L12（中英双语）直接通过 `transformers` 离线加载
- 解决 SentenceTransformers 5.x 忽略离线标志的问题
- `--repair` 一键重建全部向量索引（脚本 + ChromaDB 技能/知识库）
- 嵌入函数切换时自动检测不匹配并重建集合

### 4.6 上下文管理与自愈

- 上下文压缩：60 条消息 → 77 条（LLM 摘要旧轮次）
- 图像清理：5-动作窗口外的过期截图自动移除
- 双向孤儿修复：assistant tool_calls 缺响应 → 注入；tool 响应缺对应 calls → 删除
- 错位压缩修复：移除有 bug 的 `skip_actions`，始终压缩当前前 10 个动作
